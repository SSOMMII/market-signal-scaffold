## 3. 수정된 주요 버그

### 1) 예측이력 API 정확도 통계 로직 개선

- **파일**: `app/api/history/route.ts`
- **문제**: `actual` 값 계산 불가 시(daily_indicators에 당일 데이터 없음) `actual = direction`으로 강제 설정되어 `hit = true` 처리됨으로써 적중률이 실제보다 과도하게 높게 표시됨.
- **수정**:
  - `actual` 타입을 `'상승' | '하락'`에서 `'상승' | '하락' | null`로 변경.
  - `hit` 타입을 `boolean`에서 `boolean | null`로 변경.
  - 미검증 레코드(null)는 적중률 분모에서 제외: `verifiable.filter(p => p.hit !== null)` 기준으로 계산.

### 2) 예측이력 초기 데이터 노출 오류

- **파일**: `app/history/page.tsx`, `lib/historyData.ts`
- **문제**: `useState(mockPredictions)` 초기값으로 인해 더미 데이터(47건, 74.5%)가 API 응답 전 일시적으로 렌더링됨.
- **수정**:
  - 초기값을 `mockPredictions` 대신 빈 배열 `[]`로 설정.
  - `Prediction` 타입의 `actual`, `hit` 필드 null 허용으로 확장.
  - 미검증 항목: 회색 `?` 배지 + 회색 텍스트 표시.

### 3) 상세분석 페이지 인코딩 오류

- **파일**: `app/detail/page.tsx`
- **문제**: `signal === '매수'` 조건문이 VSCode Chat 작업 중 발생한 인코딩 깨짐으로 인해 항상 `false`로 작동, 매수 비중이 초록색이 아닌 회색으로 표시됨.
- **수정**: 해당 조건문을 정상적인 UTF-8 텍스트(`signal === '매수'`)로 전체 교체.
