## 수정된 버그

### 1. `_LEVERAGE3X_PATTERNS` NameError
- **파일**: `scripts/train_lgbm.py`
- **문제**: `_get_label_threshold()` 함수에서 참조하는 4개 패턴 변수가 정의되지 않음
- **원인**: VSCode Chat 작업 중 변수 정의 누락
- **수정**: `_TIER_THRESHOLDS` 아래에 패턴 집합 추가
  ```python
  _LEVERAGE3X_PATTERNS = {'SOXL', 'TQQQ', ...}
  _LEVERAGE2X_PATTERNS = {'QLD', 'SSO', '122630', ...}
  _HIGH_VOL_PATTERNS   = {'005930', '000660', ...}
  _BOND_INVERSE_PATTERNS = {'TLT', 'TBT', '114800', ...}
  ```

### 2. `ai_predictions` ON CONFLICT 오류 (42P10)
- **문제**: `add_model_version_to_predictions.sql` 마이그레이션으로 UNIQUE 제약이 `(ticker, date, model_version)`으로 변경됐으나 Python 코드는 `ticker,date`만 사용
- **수정**:
  - `on_conflict="ticker,date,model_version"` 으로 변경
  - `model_version: "A"` 필드를 upsert rows에 추가
  - Supabase SQL Editor에서 마이그레이션 수동 실행

### 3. `collect_yfinance.py` 다중 버그
- **`import os` 누락**: `NameError: name 'os' is not defined`
- **`.env.local` 미로드**: Supabase 환경변수 인식 불가 → fallback 사용
- **KR 심볼 형식 불일치**: DB에 `005930`(6자리)로 저장된 심볼을 yfinance는 `005930.KS` 형식으로 요구
- **수정**: `import os` 추가, `.env.local` 로더 추가, `_normalize_kr_symbol()` 함수 추가

### 4. `upsert_supabase.py` UTF-8 BOM 오류
- **문제**: PowerShell `Out-File -Encoding utf8`이 UTF-8 BOM(utf-8-sig) 형식으로 저장 → JSON 파싱 실패
- **수정**: `read_text(encoding="utf-8-sig")`로 변경
