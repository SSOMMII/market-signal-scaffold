## 1. 빌드 에러 수정

### 1) ESLint Circular JSON 에러 해결

- **파일**: `.eslintrc.json`, `package.json`
- **증상**: `npm run lint` 및 `npm run build` 시 `Converting circular structure to JSON` 에러 발생, `react` 플러그인 순환참조로 빌드 중단.
- **원인**: `eslint-config-next@16.2.1` (Next.js 16용, ESLint 9 flat config 전용)이 `next@14.2.35` 와 버전 불일치. 두 config 포맷이 충돌하며 플러그인 객체 순환참조 발생.
- **수정**:
  - `eslint-config-next` → `14.2.35`로 다운그레이드 (next 버전과 일치)
  - `.eslintrc.json`에서 중복 선언된 `plugin:react/recommended`, `plugin:react-hooks/recommended` 제거
  - `@typescript-eslint/recommended` + parser/plugin만 유지
  - `caughtErrorsIgnorePattern: "^_"` 추가 (catch 변수 ignore 패턴)
- **결과**: `npm run lint` Error 0개 (Warning만 남음)

### 2) Migration API `exec_sql` RPC 에러 해결

- **파일**: `app/api/migrate/route.ts`
- **증상**: 빌드 시 `Migration error: { code: 'PGRST202' }` 로그 출력. Supabase에 `exec_sql` 함수 미등록으로 DDL 실행 불가.
- **수정**: `exec_sql` RPC 방식 완전 제거 → admin 클라이언트로 `calibration_method` 컬럼 SELECT 시도 후 존재 여부 판단
  - 컬럼 있음 → `200 { success: true, message: "Migration already applied" }`
  - 컬럼 없음 → `412 { success: false, sql: "..." }` (수동 실행용 SQL 포함 응답)
- **결과**: 빌드 중 Migration error 로그 완전 제거

### 3) `lib/abTest.ts` unused variable 에러 수정

- **파일**: `lib/abTest.ts`
- **수정**: `catch (e)` → `catch (_e)` (두 곳)
- **결과**: `@typescript-eslint/no-unused-vars` Error 제거
