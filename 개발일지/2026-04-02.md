## 3. KIS 외국인 수급 파이프라인 전면 수정

### 배경
`/api/cron/kis-flow` 수동 호출 시 `{"ok":false,"error":"KIS market investor flow error 404"}` 발생.
`inquire-investor-trend-total` 엔드포인트가 실제 KIS API에 존재하지 않아 발생한 문제.

### 수정 내용

| 파일 | 변경 |
|---|---|
| `lib/collectors/kr/kis.ts` | `getKisMarketInvestorFlow` 함수 제거 (404 나는 엔드포인트 호출) |
| `app/api/cron/kis-flow/route.ts` | 종목별 `inquire-investor` (FHKST01010900) 5개 종목 순차 조회 + 합산 방식으로 전면 교체 |
| `app/api/cron/kis/route.ts` | `frgn_ntby_qty` → `frgn_ntby_tr_pbmn` (거래대금 원화), 기관/개인 필드도 수집 추가, 종목 간 딜레이 300ms 추가 |
| `app/api/foreign-flow/route.ts` | `.neq('net_buy', 0)` 조건 추가로 0짜리 행 스킵 |
| `db/schema.sql` | `daily_indicators`에 `orgn_net_flow`, `prsn_net_flow` 컬럼 추가 |
| `lib/supabaseClient.ts` | `getDailyIndicatorsBySymbol` SELECT에 3개 수급 컬럼 추가 |

### 디버깅 과정에서 발견한 KIS API 특이사항

- **응답 키**: `output1` 아님 → **`output`** (배열)
- **당일 row**: `output[0]`은 장중/미확정 → 수급 필드 전부 빈 문자열 `""`
  - 해결: `find(r => r.frgn_ntby_tr_pbmn !== '' && r.frgn_ntby_tr_pbmn != null)` 로 첫 유효 행 사용
- **단위**: `frgn_ntby_tr_pbmn` = 백만원 단위 → 원화로 `× 1e6` 필요
- **개인 필드명**: `indvdl_ntby_tr_pbmn` 아님 → **`prsn_ntby_tr_pbmn`**

### Supabase 마이그레이션 (수동 실행 완료)

```sql
ALTER TABLE daily_indicators
  ADD COLUMN IF NOT EXISTS orgn_net_flow numeric(24, 4),
  ADD COLUMN IF NOT EXISTS prsn_net_flow numeric(24, 4);
```
