### ✅ 구현된 개선사항

#### 1. MACD 모멘텀 가속도 감지
- **문제점**: 기존에는 골든크로스 방향만 감지
- **개선안**: 히스토그램의 가속/감속 패턴 감지
- **구현 내용**:
  - `macd_hist_accel`: MACD 히스토그램의 전일 대비 변화율
  - `macd_cross`: 골든크로스(+1)/데드크로스(-1) 방향 유지
- **피처 추가**: `macd_hist_accel`, `macd_cross`

#### 2. Bollinger Band 4구간 점수화
- **문제점**: 5%/95% 극단 구간만 반응
- **개선안**: 15%/85% 중간 구간도 절반 점수 부여
- **구현 내용**:
  - `bb_zone_score`: 4구간 점수화 시스템
    - 0-15%: -1.0점 (극단적 저점)
    - 15-50%: -0.5 ~ 0점 (약한 저점)
    - 50-85%: 0 ~ +0.5점 (약한 고점)
    - 85-100%: +1.0점 (극단적 고점)
- **피처 추가**: `bb_zone_score`

#### 3. SMA 이격도 검토
- **결과**: 이미 `(close - sma) / sma` 비율로 연속 점수화 구현되어 있었음
- **개선 필요 없음**: 기존 로직 유지

## 타입 오류 수정 및 빌드 안정화

### ✅ 해결된 오류

#### 1. update-returns API 타입 오류
- **문제점**: `app/api/cron/update-returns/route.ts:58`에서 `.in()` 메소드에 Supabase 쿼리 빌더 객체 전달
- **오류 내용**: `PostgrestFilterBuilder` 타입을 `readonly any[]`로 변환할 수 없음
- **해결 방법**:
  - `.in()` 대신 `.eq()` 사용으로 변경
  - 먼저 `market_master`에서 `id`를 조회한 후 직접 사용
  - 타입 안전성 확보 및 쿼리 성능 개선
- **코드 변경**:
  ```typescript
  // 수정 전 (오류)
  .in('market_master_id', supabase.from('market_master').select('id').eq('symbol', row.etf_code))
  
  // 수정 후 (정상)
  const { data: marketMasterData } = await supabase.from('market_master').select('id').eq('symbol', row.etf_code).maybeSingle()
  .eq('market_master_id', marketMasterData.id)
  ```
