### 5. LightGBM 학습용 과거 데이터 수집

#### 배경
- 기존 `calcScore()`는 RSI/MACD/SMA 임계값을 사람이 직접 정한 규칙 기반 방식
- LightGBM 도입을 위해 최소 1~2년치 과거 데이터 필요 → 5년치 수집 결정

#### 버그 수정: `scripts/collect_yfinance.py`
- `collect_by_date()` 내 날짜 필터 오류 발생
  - 원인: `hist.index >= pd.Timestamp(start_date)` — yfinance가 반환하는 index가 timezone-aware(`America/New_York`, `Asia/Seoul` 등)인데 `pd.Timestamp`는 timezone-naive라 비교 실패
  - 수정: `hist.index.date()` 기반 date 레벨 비교로 교체 → 타임존 무관하게 동작

#### 실행
```bash
python collect_yfinance.py 2021-01-01 2026-03-27 | python upsert_supabase.py --mode yfinance
```

#### 결과
- 21개 종목 × 5년치 (2021-01-01 ~ 2026-03-27)
- `daily_indicators` 테이블에 **27,663건** upsert 완료
- 수집 종목: ETF 7개(QQQ/SPY/IWM/GLD/TLT/SOXL/TQQQ) + INDEX 6개 + FUTURES 3개 + FX 3개 + COMMODITY 2개
- 포함 지표: RSI / MACD / SMA50/120/200 / 볼린저밴드 / 스토캐스틱

### 6. LightGBM 학습 스크립트 (`scripts/train_lgbm.py`)

#### 피처 설계
기술적 지표만 쓰면 서로 상관관계가 높아 정보량이 적음 → 3가지 카테고리로 피처 구성

| 카테고리 | 피처 |
|---------|------|
| 가격 모멘텀 | return_1d, return_5d, return_20d, vol_20d, vol_ratio, hl_range |
| 기술적 지표 | rsi, macd_hist, stoch_k, bb_position, price_vs_sma50/120/200 |
| 크로스-에셋 | vix_close, vix_ret_1d, gold_ret_1d/5d, usdkrw_ret_1d, tnx_ret_1d/5d |

#### 레이블 정의
- 기준: 5거래일 후 수익률
- 매수(1): +2% 이상 / 매도(-1): -2% 이하 / 관망(0): 그 사이
- 레버리지 ETF(SOXL/TQQQ)는 임계값 별도 적용 (±4~5%)

#### 학습 방식
- **TimeSeriesSplit 5-fold** 교차검증 — 미래 데이터 누수 방지 (random split 사용 금지)
- fold 간 gap=5 적용 (레이블 생성 기간과 동일)
- LightGBM multiclass (3분류: 매수/관망/매도)

#### 학습 결과
```
CV Accuracy: 52.7% ± 6.3%  (랜덤 33% 대비 +20%p)
폴드별: [42.8%, 51.1%, 57.2%, 50.9%, 61.4%]
```

#### 피처 중요도 (상위 5)
| 순위 | 피처 | 의미 |
|------|------|------|
| 1 | gold_ret_5d | 금 5일 수익률 (안전자산 선호) |
| 2 | tnx_ret_5d | S&P500 5일 수익률 |
| 3 | vix_close | 공포지수 절대값 |
| 4 | gold_ret_1d | 금 1일 수익률 |
| 5 | usdkrw_ret_1d | 달러 강세/약세 |

→ **크로스-에셋 피처가 기술적 지표보다 훨씬 중요** (macd_hist 7위, rsi 10위 밖)

|------|------|----------|
| GLD | BUY | 59% |

| IWM | HOLD | 26% |

### 7. GitHub Actions 일일 자동 예측 크론 (`.github/workflows/lgbm-predict.yml`)

#### 배경
- Vercel 크론은 Next.js API route만 지원 → Python 스크립트 실행 불가
- GitHub Actions 무료 티어 활용 (2000분/월)

#### 동작 방식
```
매일 평일 22:00 UTC (한국 오전 7시)
  └─ 모델 캐시 있음? → Yes → predict-only 실행 (~30초)
                    → No  → 전체 재학습 + 예측 (~5분)
  └─ ai_predictions 테이블에 7개 ETF 예측 upsert
```

#### 구현 포인트
- `actions/cache@v4`로 `scripts/models/lgbm_model.pkl` 캐싱
  - 캐시 키: `train_lgbm.py` 파일 해시 → 스크립트 변경 시 자동 재학습
- 수동 실행 시 `train` / `predict-only` 모드 선택 가능 (`workflow_dispatch`)
- GitHub Secrets: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` 등록

#### 설정 과정
1. PAT에 `workflow` scope 추가 후 push
2. `main` 브랜치가 원격 기본 브랜치(`master`)와 달라 Actions 미인식 → `git push origin main:master`로 동기화
3. GitHub Secrets 2개 등록 (`SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`)
4. 수동 실행(Run workflow) → **Status: Success, 56초** 확인

- 데이터 없는 종목 → 에러 대신 "기술 지표 데이터 없음" 안내 화면

#### 신규 API: `app/api/analysis/[symbol]/route.ts`

| 데이터 | 소스 |
|--------|------|
| RSI / MACD / SMA50 | Supabase `daily_indicators` (KIS 실데이터) |
| AI Score / 신호 | RSI+MACD+SMA 룰 기반 계산 (-4.5 ~ +4.5) |
| Fear & Greed | Alternative.me 무료 API |
| Reddit 언급량 | ApeWisdom 무료 API (US 종목만) |
| 주요 시그널 텍스트 | 실제 지표값 해석 자동 생성 |

**AI Score 로직**
- RSI < 30 → +2.0 / RSI > 70 → -2.0 / 중간 ±0.5
- MACD > Signal → +1.5 / MACD < Signal → -1.5
- Close > SMA50 → +1.0 / Close < SMA50 → -1.0
- 매수: ≥ +1.5 / 매도: ≤ -1.5 / 관망: 그 사이

**버그 수정 (검토 후)**
- MACD signalLine null 시 0과 비교하던 버그 수정
- "동종업계 대비" 오해 문구 → "기술지표 종합 강세/중립/약세"로 교체
- Fear & Greed에 "(시장 전체)" 라벨 추가
